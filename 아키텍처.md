# AgentDeck 아키텍처

AI 에이전트(Claude Code, Gemini CLI, Codex CLI 등)를 브라우저에서 관리하는 터미널 대시보드.
Go 단일 바이너리에 React 프론트엔드가 임베드되어, 외부 의존성 없이 실행된다.

---

## 시스템 개요

```
Browser (React SPA)
    |
    |  REST API (HTTP)  +  WebSocket (실시간)
    v
Go Server (단일 바이너리)
    |
    +-- SQLite (에이전트/프로젝트/로그 메타데이터)
    +-- tmux (에이전트 세션 관리)
    +-- PTY (터미널 I/O)
    +-- fsnotify (파일 변경 감시)
```

## 서버 (Go)

### 초기화 흐름

```
main.go
  1. config.Load()         -- .env에서 PIN/JWT/PORT/DB 로드 (첫 실행 시 자동 생성)
  2. db.Init()             -- SQLite 초기화 + 마이그레이션
  3. 서비스 계층 생성       -- TmuxSvc, PtySvc, AgentSvc, FileSvc, WatcherSvc, ProjectSvc, AuthSvc
  4. ws.NewHub()           -- WebSocket 허브 (PtySvc, WatcherSvc, AgentSvc 의존)
  5. 라우터 구성            -- Gorilla Mux + 미들웨어 + 핸들러 + 정적 파일 서빙
  6. 서버 시작 + 브라우저 열기
```

### DB 스키마

```sql
-- agents: AI 에이전트 정의
CREATE TABLE agents (
    id TEXT PRIMARY KEY,                    -- 8자리 hex
    preset TEXT NOT NULL,                   -- 'claude-code', 'gemini-cli', etc.
    name TEXT NOT NULL,
    tmux_session TEXT NOT NULL UNIQUE,      -- "agentdeck-{id}"
    working_dir TEXT NOT NULL,
    command TEXT NOT NULL,
    args TEXT DEFAULT '[]',                 -- JSON 배열
    status TEXT DEFAULT 'stopped',          -- 'running' | 'stopped'
    color_hue INTEGER DEFAULT 220,          -- HSL Hue (0-360)
    color_name TEXT DEFAULT 'blue',
    created_at TEXT,
    updated_at TEXT
);

-- recent_projects: 최근 열었던 프로젝트
CREATE TABLE recent_projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    path TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    last_opened_at TEXT,
    last_agent_preset TEXT,
    open_count INTEGER DEFAULT 1
);

-- logs: 에이전트 활동 로그 (FTS5 전문 검색)
CREATE TABLE logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_id TEXT NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    data TEXT NOT NULL,
    created_at TEXT
);
CREATE VIRTUAL TABLE logs_fts USING fts5(data, content='logs', content_rowid='id');
```

### 인증 흐름

```
Client                          Server
  |                               |
  |-- POST /api/auth/login ------>|  PIN 6자리 검증
  |   { pin: "847293" }          |
  |<-- { accessToken, ----------|  JWT 발급 (access: 7일, refresh: 30일)
  |      refreshToken }          |
  |                               |
  |-- Authorization: Bearer ... ->|  JWT 검증 (모든 API)
  |-- /ws?token=... ------------>|  JWT 검증 (WebSocket)
  |                               |
  |-- POST /api/auth/refresh ---->|  access 만료 시 갱신
```

Rate limit: 인증 엔드포인트 10 req/min (IP 기반)

### 미들웨어 체인

```
요청 → Helmet → CORS → [RateLimiter (인증만)] → [Auth (API만)] → Handler
```

- **Helmet**: CSP, X-Frame-Options, X-Content-Type-Options 등
- **CORS**: 설정된 origin 허용, preflight 처리
- **RateLimiter**: IP별 슬라이딩 윈도우
- **Auth**: Bearer 토큰 검증

### WebSocket 이벤트

```
Client → Server (inbound)
  terminal:attach   { agentId, cols, rows }  -- PTY 접속
  terminal:detach   {}                        -- PTY 해제
  terminal:input    { agentId, data }         -- 키보드 입력
  terminal:resize   { agentId, cols, rows }   -- 터미널 리사이즈
  file:watch        { agentId, path }         -- 파일 감시 시작
  file:unwatch      { agentId, path }         -- 파일 감시 중지

Server → Client (outbound)
  terminal:output   { agentId, data }         -- PTY 출력
  agent:created     Agent                     -- 에이전트 생성됨
  agent:destroyed   { agentId }               -- 에이전트 삭제됨
  agent:status      { agentId, status }       -- 상태 변경
  agent:list        Agent[]                   -- 에이전트 목록
  file:changed      { path, operation }       -- 파일 변경 감지
```

연결 유지: ping/pong 54초 주기, 메시지 최대 1MB, 쓰기 타임아웃 10초

### 데이터 흐름

#### 터미널 출력 → Sub-agent 시각화

```
Go Server (PTY ReadPump)
  → WebSocket: terminal:output { agentId, data }
    → TerminalView: xterm.write(data)                  -- 터미널 렌더링
    → useSubAgents: processOutput(data)                 -- 패턴 매칭
      → 정규식으로 도구 감지 (Read/Write/Bash/Search/Think/Agent/Web/Diff)
        → addSubAgent() → Zustand store 업데이트
          → SubAgentBar 리렌더링 (PixelSprite 애니메이션)
          → SubAgentPanel 리렌더링 (Orbital + Timeline)
          → playToolSound() (사운드 활성화 시)
        → 8초 후 자동 완료, 15초 후 cleanup
```

#### 에이전트 생명주기

```
Create: POST /api/agents → tmux 세션 생성 → DB 저장 → WS broadcast agent:created
Attach: WS terminal:attach → PTY 접속 (tmux attach) → ReadPump 시작
Input:  WS terminal:input → PTY write
Output: PTY read → WS terminal:output → xterm.write
Delete: DELETE /api/agents/{id} → PTY 종료 → tmux kill → DB 삭제 → WS broadcast agent:destroyed
```

### 애니메이션 시스템

```
PixelSprite
  ├─ 16x16 SVG 픽셀 그리드
  ├─ 프리셋별 idle/active 프레임 (4fps/8fps)
  ├─ 에이전트 colorHue → paletteGenerator → 색상 배열
  └─ 테마: default / cat (catPresets.ts)

SubAgentParticles
  └─ Canvas 기반, active 도구별 색상 파티클, requestAnimationFrame 루프

CSS 애니메이션
  └─ orbital-spin, glow-pulse, slide-in-right, slide-up 등
```

### 사운드 시스템

Web Audio API 기반 레트로 게임 스타일 효과음:
- 에이전트 시작/종료
- Sub-agent 생성/완료/에러
- 도구별 고유 사운드 (Read=페이지 넘김, Write=필기, Bash=키보드, Search=소나, Think=허밍)

## 빌드

```
Makefile
  make setup    -- pnpm install (client)
  make build    -- pnpm build (client) → go build -o agentdeck (임베드)
  make dev      -- 개발 서버 (hot reload)
```

최종 산출물:
- `agentdeck` -- Go 바이너리 1개 (React SPA 임베드)
- `agentdeck.db` -- SQLite DB 파일
- `.env` -- 설정 (자동 생성)
